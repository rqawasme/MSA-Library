# CLAUDE.md - MSA-Library

## Project Overview

MSA-Library is a Django-based library management system for the SFU (Simon Fraser University) Muslim Students Association. It allows members to borrow and return books, with admin features for inventory management and overdue email reminders.

**Live site:** https://sfumsa-library.onrender.com

---

## Tech Stack

- **Backend:** Django 3.2.16, Django REST Framework 3.14.0
- **Database:** PostgreSQL
- **Frontend:** Django Templates, W3.CSS, SCSS/SASS (django-sass-processor)
- **Server:** Gunicorn (production), Django dev server (local)
- **Hosting:** Render.com
- **Email:** SMTP via sfumsa.ca

## Development Setup

### Prerequisites
- Python 3.x
- PostgreSQL database
- `.env` file with required variables (see below)

### Running Locally

```bash
pip install -r requirements.txt
python manage.py migrate
python manage.py loaddata superuser_fixture.json
python manage.py runserver
```

The build script `./build.sh` also handles installs, migrations, fixture loading, and static file collection.

### Required Environment Variables

Create a `.env` file in the project root:

```
DJANGO_SECRET_KEY=<your-secret-key>
DATABASE_NAME=<db-name>
DATABASE_USER=<db-user>
DATABASE_PASSWORD=<db-password>
DATABASE_HOST=<db-host>
DATABASE_PORT=5432
EMAIL_HOST_USER=<email@sfumsa.ca>
EMAIL_HOST_PASSWORD=<email-password>
```

## Key Models

**`accounts.User`** — Custom user model; uses email as username.
- Fields: `first_name`, `last_name`, `email`, `phone_number`, `is_staff`, `is_superuser`

**`library.Book`** — Represents a physical book in the library.
- Fields: `title`, `author`, `description`, `available` (bool), `unique_number` (int, unique)

**`library.Signout`** — Tracks a book loan.
- Fields: `book` (FK), `user` (FK), `signout_date`, `signin_date` (nullable), `signed_back_in` (bool), `expected_return_date`
- Default loan period: **21 days**

## URL Routes

### Web UI

| URL | View | Access |
|-----|------|--------|
| `/` | Borrow book (SignoutView) | Authenticated |
| `/signout/` | Borrow book (duplicate) | Authenticated |
| `/signin/` | Return book (SigninView) | Authenticated |
| `/books/` | Browse all books | Authenticated |
| `/addbook/` | Add new book | Admin only |
| `/history/` | View all borrowed books | Admin only |
| `/overdue/` | View overdue books | Admin only |
| `/sendreminders/?keyword=alwayshalalmemes` | Trigger overdue emails | Admin only |
| `/admin/` | Django admin panel | Admin only |
| `/accounts/signup/` | Register | Public |
| `/accounts/login/` | Login | Public |

### REST API

| URL | Description |
|-----|-------------|
| `/booklist/` | List/create books |
| `/booklist/{id}/` | Retrieve/update/delete book |
| `/signoutlist/` | List/create signouts |
| `/signoutlist/{id}/` | Retrieve/update/delete signout |

## Business Logic Notes

- A book's `available` flag is flipped to `False` when borrowed and back to `True` when returned.
- If a book is marked unavailable manually, all prior signouts for it are marked as returned.
- Overdue emails are sent via `library/tasks.py` — triggered by a cron job or the `/sendreminders/` endpoint.
- Emails in development are stored as files in `BASE_DIR/sent_emails/`.

## Deployment (Render.com)

Render runs `./build.sh` on build, then starts with:

```
gunicorn msalibrary.wsgi:application
```

Static files are collected to `/opt/render/project/src/static/`.

`DJANGO_SECRET_KEY` is auto-generated by Render on first deploy.

- `LOGIN_REDIRECT_URL = "home"`
- `LOGOUT_REDIRECT_URL = "/accounts/login/"`
